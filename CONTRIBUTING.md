# Contributing

Thanks for considering a contribution to `claude-code-pr-review-respond`.

## Ways to contribute

- **Bug fixes** — the skill doesn't work as described in a specific scenario
- **New features** — support for additional platforms (GitLab, Gitee), new workflows
- **Documentation** — clearer instructions, additional language translations
- **Examples** — new fix patterns for different tech stacks

## Development workflow

1. Fork the repository
2. Create a feature branch: `git checkout -b feat/my-change`
3. Make your changes
4. Bump the version in `SKILL.md` frontmatter (`version: X.Y.Z`)
5. Add a dated entry to `CHANGELOG.md`
6. Update `README.md` / `README.zh-CN.md` if features, usage, or requirements change
7. Submit a pull request

## Style guide

- **Writing style**: imperative/infinitive form (e.g. "Fetch all review comments", not "You should fetch...")
- **SKILL.md**: keep it generic — project-specific patterns go in `examples/`
- **Commits**: follow the conventional commit format documented in [`references/commit-conventions.md`](references/commit-conventions.md)
- **Replies**: all GitHub comment replies generated by the skill must be in Chinese

## Pull request checklist

- [ ] SKILL.md frontmatter version bumped
- [ ] CHANGELOG.md updated
- [ ] README sync checked
- [ ] New patterns added to `examples/` if applicable
- [ ] All markdown links verified
